Basics

In the following example, there are four organizations that are equal to each other in every aspect, except they use different strategies to motivate their employees. In Existing Model, since this difference is ignored, each organization will have the same Total Income.

New Model introduces Employee Satisfaction Index. By using this index, we can enter which organization's employees satisfied with their organization's strategy and based on this new data, organizations' Total Income will now differ.

  • In Existing Model, increase organizations' Number of Sales by clicking + button.
    This will increase Number of Sales of all organizations in both models to let you compare the results easily.
  • In New Model, update Employee Satisfaction rating of the organizations by using + and - buttons next to them.
    See the difference in Total Income fields in both models.
  • CMRP Rate / Amount: Since New Model uses CMRP, it requires an amount to be distributed.
    This amount will automatically be calculated based on CMRP Rate which is 10% of the total spendings in the system.
    When you change the values of Number of Sales, CMRP Amount will be changed accordingly.
    This approach will be used throughout the samples and the details will be explained in further chapters.
  • Ratings switch: In New Model, notice Ratings switch next to its title.
    The number shows how many users have already entered their ratings for this CMRP.
    In each CMRP, toggle the switch to see only your ratings or the average of all users.

If we could rate 'Employee Satisfaction' of the organizations in real life, we could reward the ones with a higher rating as in this sample. Adding indexes to the system sets additional priorities for the organizations, instead of just focusing to profit.

Although only three different indexes will be presented in the following chapters, as long as they are measurable, more indexes can be added into the system.
